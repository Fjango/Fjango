# 🔧 站长操作手册（私密，不要发到网站）

这份文档只有你能看。记录了你管理 Fjango 网站的所有操作。

---

## 📬 如何处理投稿人的投稿

### 有人通过邮箱/QQ/微信投稿了，你收到了一篇文章

**第1步：** 打开 `docs\share\` 文件夹

**第2步：** 新建一个 `.md` 文件（复制 `sample-article.md` 然后改名最简单）
- 文件名用英文或拼音，比如 `xiaoming-java-experience.md`

**第3步：** 用记事本打开，把投稿内容填进去，格式如下：

```markdown
# 投稿人给的标题

!!! info "作者信息"
    - **作者**：投稿人的名字
    - **时间**：2026年X月X日

## 正文

（这里粘贴投稿人的内容）
```

**第4步：** 把文章加到导航栏。打开 `mkdocs.yml`，在 `经验分享:` 下面加一行：

```yaml
  - 经验分享:
    - 目录: share/index.md
    - 文章的显示名称: share/文件名.md   ← 加这一行
```

**第5步：** 告诉我：

> "帮我推送一下，新加了文章"

我帮你推到网站。或者你也可以自己推送（见下方"怎么推送"）。



---

## ✍️ 你自己要发表文章

跟上面处理投稿一样，只是内容是你自己写的。

1. 在 `docs\share\` 新建 `.md` 文件
2. 写内容
3. 在 `mkdocs.yml` 加导航链接
4. 告诉我推送



---

## 🗑️ 怎么删除文章

### 完全删掉

1. 在 `docs\share\` 里删掉那个 `.md` 文件
2. 打开 `mkdocs.yml`，删掉那篇文章的导航链接
3. 告诉我推送

### 只是从导航里隐藏（暂时下线）

1. 打开 `mkdocs.yml`，在那行前面加一个 `#`（注释掉）
2. 告诉我推送



---

## ✏️ 怎么修改文章

1. 在 `docs\share\` 里找到那个文件，用记事本打开
2. 直接改内容
3. 告诉我推送



---

## 🚀 怎么自己推送（不用每次叫我）

在电脑上打开终端（命令提示符），依次输入三行：

```bash
cd C:\Users\PC\fjango
git add -A
git commit -m "更新内容"
git push origin main
```

推送完等1-2分钟，网站就自动更新了。

> ⚠️ 如果推送失败，把错误信息截图给我。

## 📁 项目结构速览

```
fjango\
  ├── docs\
  │   ├── index.md              ← 首页
  │   ├── about.md              ← 关于页
  │   ├── contribute-guide.md   ← 投稿指南（公开的）
  │   ├── share\
  │   │   ├── index.md          ← 经验分享目录页
  │   │   └── 各种文章.md        ← 所有文章在这里
  │   └── assets\               ← 图片放这里
  ├── mkdocs.yml                ← 网站配置 + 导航栏
  ├── ADMIN-GUIDE.md            ← 这个文件（只有你看到）
  └── README.md
```
